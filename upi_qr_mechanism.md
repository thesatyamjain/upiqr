# UPI QR Code Generation & Mobile Share Mechanism

This document provides a comprehensive reference for generating Unified Payments Interface (UPI) deep links, dynamically rendering a premium 3:4 aspect ratio payment card in the browser using HTML5 Canvas, and sharing the generated card on mobile devices using the Web Share API (along with fallback mechanisms).

---

## 1. UPI Deep Linking Specification

UPI uses a standard custom URI scheme (`upi://pay`) to launch UPI-compliant payment apps (such as Google Pay, PhonePe, Paytm, BHIM, etc.) on mobile devices. When a user clicks a UPI deep link or scans a QR code containing the link, the host OS passes the intent to the registered payment service provider (PSP) apps.

### Standard URI Format
```text
upi://pay?pa=payee@bank&pn=Payee%20Name&am=100.00&cu=INR&tn=Invoice%20Payment
```

### Query Parameters

| Parameter | Name | Type | Description | Max Length | Requirement |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **`pa`** | Payee Address (VPA) | String | The Virtual Payment Address (UPI ID) of the recipient. Must be validly formatted (e.g., `username@bank`). | 255 | **Mandatory** |
| **`pn`** | Payee Name | String | The legal name of the recipient or merchant. Must match the name registered with the bank. | 99 | **Mandatory** |
| **`am`** | Amount | Decimal | The transaction amount. Must be formatted with up to two decimal places (e.g., `150.00`). If absent, the user can enter any amount in the payment app. | 10 | Optional |
| **`cu`** | Currency | String | The currency code. Currently, only Indian Rupee (`INR`) is supported. Defaults to `INR` if omitted. | 3 | Optional |
| **`tn`** | Transaction Note | String | A short description, remark, or transaction note. Often shown in transaction history. | 80 | Optional |
| **`tr`** | Transaction Ref ID | String | Unique transaction reference ID generated by the merchant system (e.g., order ID, billing ID). | 35 | Optional (Mandatory for merchants) |
| **`tid`** | Transaction ID | String | PSP-generated transaction ID (identifies the transaction on the gateway). | 35 | Optional (Mandatory for merchants) |
| **`mc`** | Merchant Category Code | String | A 4-digit Merchant Category Code (MCC) as defined by ISO 18245 (e.g., `5411` for groceries). | 4 | Optional (Mandatory for merchants) |
| **`mam`** | Minimum Amount | Decimal | The minimum amount that must be paid. If specified, the user cannot pay less than this amount. | 10 | Optional |
| **`url`** | Reference URL | String | An optional web URL providing transaction details, invoice copy, or verification metadata. | 255 | Optional |
| **`mode`** | Initiation Mode | String | Indicates how the transaction was initiated. Standard 2-digit codes:<br>`01` - Static QR code<br>`02` - Dynamic QR code<br>`04` - Intent / App-to-App<br>`05` - Secure Intent | 2 | Optional |
| **`orgid`** | Originating PSP ID | String | The unique identifier of the acquiring bank or payment gateway. | 20 | Optional (Mandatory if `mode` is secure) |
| **`sign`** | Digital Signature | String | Base64-encoded cryptographic signature generated using the merchant's private key to prevent parameter tampering. | 256 | Optional (Mandatory if `mode` is secure) |

### Validation and Encoding Rules
1. **URL Encoding**: All query parameters must be properly URL-encoded (e.g., space characters converted to `%20` or `+`, special symbols encoded).
2. **Amount Format**: The `am` parameter must be numeric, positive, and should not exceed the daily transaction limit of the user's account or app (generally ₹1,000,000 for regular P2P/P2M, or higher for specific categories like education and healthcare).
3. **No Unknown Parameters**: The URI must not contain custom query parameters outside the official NPCI spec, as some payment apps will reject the link as invalid.

## 2. Dynamic Premium 3:4 Payment Card Generation (HTML5 Canvas)

To generate a premium payment card image, we target a **3:4 aspect ratio** (e.g., `600px` width by `800px` height). This is perfect for mobile screens, sharing on WhatsApp, or downloading to the camera roll.

### Key Canvas Rendering Phases
1. **Backdrop & Rounded Corners**: Paint a smooth background gradient (e.g., deep indigo `#0f172a` to `#1e1b4b`). Draw rounded corners using `ctx.roundRect` or arc clips to represent a card edge.
2. **Branding Header**: Draw a custom vector "UPI QR" branding using canvas graphics (paths, arcs, and gradients) to avoid external image loading bottlenecks and CORS issues.
3. **QR Container (Safe Zone)**: Draw a clean white container with rounded corners and a soft shadow (using `ctx.shadowColor` and `ctx.shadowBlur`). This acts as the high-contrast scanning background.
4. **QR Code Generation**: Use a standard client-side library (`qrcode.js` or `node-qrcode`) to render the `upi://pay` URI as a QR code image, then paint it onto the canvas coordinate grid.
5. **Payee Details**: Render the recipient's legal name (`pn`) and UPI ID (`pa`) using crisp, anti-aliased sans-serif fonts.
6. **Amount Badge**: If a transaction amount is specified, render a premium badge displaying the amount in green; if not, render a "Scan to Pay Any Amount" placeholder.

### Reference Canvas Generator Code
```javascript
/**
 * Renders a premium 3:4 payment card to a canvas element.
 * 
 * @param {HTMLCanvasElement} canvas - The target canvas element.
 * @param {Object} data - Payment details (pa, pn, am, tn).
 * @returns {Promise<void>} Resolves when rendering is complete.
 */
async function renderUPIPaymentCard(canvas, data) {
  const ctx = canvas.getContext('2d');
  
  // Set dimensions (3:4 ratio)
  canvas.width = 600;
  canvas.height = 800;

  // Clear shadow states
  ctx.shadowColor = 'transparent';
  ctx.shadowBlur = 0;
  ctx.shadowOffsetX = 0;
  ctx.shadowOffsetY = 0;

  // 1. Draw Background Gradient
  const bgGrad = ctx.createLinearGradient(0, 0, 0, canvas.height);
  bgGrad.addColorStop(0, '#1e1b4b'); // Deep Royal Blue-Indigo
  bgGrad.addColorStop(1, '#090514'); // Very Dark Violet
  ctx.fillStyle = bgGrad;
  ctx.fillRect(0, 0, canvas.width, canvas.height);

  // 2. Draw Decorative Vector Background Elements
  ctx.fillStyle = 'rgba(99, 102, 241, 0.15)'; // Semi-transparent Indigo
  ctx.beginPath();
  ctx.arc(0, 0, 250, 0, Math.PI * 2);
  ctx.fill();

  ctx.beginPath();
  ctx.arc(canvas.width, canvas.height * 0.4, 180, 0, Math.PI * 2);
  ctx.fill();

  // 3. Draw Main Card Panel (Glassmorphism look)
  const cardX = 30;
  const cardY = 30;
  const cardW = canvas.width - (cardX * 2);
  const cardH = canvas.height - (cardY * 2);
  const radius = 24;

  ctx.save();
  ctx.fillStyle = 'rgba(255, 255, 255, 0.04)';
  ctx.strokeStyle = 'rgba(255, 255, 255, 0.12)';
  ctx.lineWidth = 1.5;
  ctx.beginPath();
  ctx.roundRect(cardX, cardY, cardW, cardH, radius);
  ctx.fill();
  ctx.stroke();
  ctx.restore();

  // 4. Draw Branding Header (Unified Payments Interface styling)
  ctx.save();
  ctx.font = 'bold 22px system-ui, -apple-system, sans-serif';
  ctx.fillStyle = '#ffffff';
  ctx.letterSpacing = '1px';
  ctx.fillText('UPI QR PAY', cardX + 30, cardY + 50);

  // Draw a vector representation of a payment badge
  ctx.fillStyle = '#22c55e'; // Green dot
  ctx.beginPath();
  ctx.arc(cardX + cardW - 40, cardY + 45, 8, 0, Math.PI * 2);
  ctx.fill();
  ctx.restore();

  // 5. Draw QR Code Container (High-contrast scanning box)
  const qrBoxSize = 300;
  const qrBoxX = (canvas.width - qrBoxSize) / 2;
  const qrBoxY = cardY + 100;
  const qrBoxRadius = 16;

  ctx.save();
  // Draw Shadow
  ctx.shadowColor = 'rgba(0, 0, 0, 0.4)';
  ctx.shadowBlur = 15;
  ctx.shadowOffsetY = 8;
  
  ctx.fillStyle = '#ffffff';
  ctx.beginPath();
  ctx.roundRect(qrBoxX, qrBoxY, qrBoxSize, qrBoxSize, qrBoxRadius);
  ctx.fill();
  ctx.restore();

  // 6. Generate and Draw QR Code inside the container
  const upiURI = constructUPIURI(data);
  const qrSize = 250;
  const qrX = qrBoxX + (qrBoxSize - qrSize) / 2;
  const qrY = qrBoxY + (qrBoxSize - qrSize) / 2;

  // Use the standard browser-side node-qrcode library (window.QRCode)
  await new Promise((resolve, reject) => {
    const tempCanvas = document.createElement('canvas');
    QRCode.toCanvas(tempCanvas, upiURI, {
      margin: 0,
      width: qrSize,
      color: {
        dark: '#0f172a',  // Very dark slate
        light: '#ffffff'  // Pure white
      }
    }, function(err) {
      if (err) return reject(err);
      ctx.drawImage(tempCanvas, qrX, qrY);
      resolve();
    });
  });

  // 7. Draw Payee Information
  ctx.save();
  ctx.textAlign = 'center';
  
  // Payee Legal Name
  ctx.fillStyle = '#ffffff';
  ctx.font = 'bold 26px system-ui, -apple-system, sans-serif';
  ctx.fillText(data.pn, canvas.width / 2, qrBoxY + qrBoxSize + 45);

  // Payee UPI ID
  ctx.fillStyle = '#94a3b8'; // Slate 400
  ctx.font = '500 16px monospace';
  ctx.fillText(data.pa, canvas.width / 2, qrBoxY + qrBoxSize + 75);
  ctx.restore();

  // 8. Draw Amount Badge
  if (data.am && parseFloat(data.am) > 0) {
    const amtString = `₹${parseFloat(data.am).toLocaleString('en-IN', { minimumFractionDigits: 2 })}`;
    
    // Measure text to size the badge dynamically
    ctx.save();
    ctx.font = 'bold 36px system-ui, -apple-system, sans-serif';
    const textWidth = ctx.measureText(amtString).width;
    const badgeW = textWidth + 40;
    const badgeH = 64;
    const badgeX = (canvas.width - badgeW) / 2;
    const badgeY = qrBoxY + qrBoxSize + 110;

    // Draw amount badge pill
    ctx.fillStyle = '#10b981'; // Emerald 500
    ctx.beginPath();
    ctx.roundRect(badgeX, badgeY, badgeW, badgeH, 32);
    ctx.fill();

    // Draw amount text
    ctx.fillStyle = '#ffffff';
    ctx.textAlign = 'center';
    ctx.textBaseline = 'middle';
    ctx.fillText(amtString, canvas.width / 2, badgeY + (badgeH / 2));
    ctx.restore();
  } else {
    // Render "Pay Any Amount" pill
    ctx.save();
    const badgeW = 260;
    const badgeH = 40;
    const badgeX = (canvas.width - badgeW) / 2;
    const badgeY = qrBoxY + qrBoxSize + 120;

    ctx.fillStyle = '#334155'; // Slate 700
    ctx.beginPath();
    ctx.roundRect(badgeX, badgeY, badgeW, badgeH, 20);
    ctx.fill();

    ctx.fillStyle = '#cbd5e1'; // Slate 300
    ctx.font = 'bold 13px system-ui, -apple-system, sans-serif';
    ctx.letterSpacing = '0.5px';
    ctx.textAlign = 'center';
    ctx.textBaseline = 'middle';
    ctx.fillText('SCAN TO PAY ANY AMOUNT', canvas.width / 2, badgeY + (badgeH / 2));
    ctx.restore();
  }

  // 9. Draw Transaction Note (if provided)
  if (data.tn) {
    ctx.save();
    ctx.textAlign = 'center';
    ctx.fillStyle = '#64748b'; // Slate 500
    ctx.font = 'italic 15px system-ui, -apple-system, sans-serif';
    ctx.fillText(`"${data.tn}"`, canvas.width / 2, canvas.height - 90);
    ctx.restore();
  }

  // 10. Draw Footer Branding & Instructions
  ctx.save();
  ctx.textAlign = 'center';
  ctx.fillStyle = '#94a3b8';
  ctx.font = '14px system-ui, -apple-system, sans-serif';
  ctx.fillText('Scan with GPay, PhonePe, Paytm, BHIM, or any UPI App', canvas.width / 2, canvas.height - 50);
  ctx.restore();
}

/**
 * Builds the official URL-encoded UPI payment string.
 */
function constructUPIURI(data) {
  const params = new URLSearchParams();
  params.append('pa', data.pa);
  params.append('pn', data.pn);
  if (data.am && parseFloat(data.am) > 0) {
    params.append('am', parseFloat(data.am).toFixed(2));
  }
  params.append('cu', 'INR');
  if (data.tn) {
    params.append('tn', data.tn);
  }
  return `upi://pay?${params.toString()}`;
}
```

## 3. Mobile-First Sharing (Web Share API & Fallbacks)

To allow the user to share the generated image directly to platforms like WhatsApp or Slack on mobile, we leverage the **Web Share API (Level 2)**. The API allows passing an array of `File` objects.

### Step-by-Step Implementation Flow
1. **Convert Canvas to Blob**: Call `canvas.toBlob(...)` to get the image binary.
2. **Convert Blob to File**: Wrap the blob in a `new File([blob], 'filename.jpg', { type: 'image/jpeg' })`.
3. **Validate Share Compatibility**: Use `navigator.canShare({ files: [file] })` to confirm if sharing the file is supported.
4. **Trigger Share Sheet**: Call `navigator.share(...)` containing the file array.

### Complete Share/Fallback Code Block
```javascript
/**
 * Shares the card canvas image via the Web Share API if supported.
 * Falls back to a browser download or a visual preview modal.
 * 
 * @param {HTMLCanvasElement} canvas - The source canvas.
 * @param {string} payeeName - The payee name (used for filename).
 */
async function shareOrDownloadCard(canvas, payeeName) {
  const safeFilename = `${payeeName.replace(/[^a-z0-9]/gi, '_').toLowerCase()}_upi_qr.jpg`;

  canvas.toBlob(async (blob) => {
    if (!blob) {
      alert('Could not capture the payment card image.');
      return;
    }

    const file = new File([blob], safeFilename, { type: 'image/jpeg' });
    const shareData = {
      files: [file],
      title: 'UPI Payment QR',
      text: `Scan and pay to ${payeeName}`
    };

    // 1. Web Share API Flow (Mobile browsers)
    if (navigator.canShare && navigator.canShare({ files: [file] })) {
      try {
        await navigator.share(shareData);
        console.log('Share completed successfully');
      } catch (err) {
        if (err.name !== 'AbortError') {
          console.error('Sharing failed:', err);
          fallbackDownload(blob, safeFilename);
        }
      }
    } else {
      // 2. Fallback Flow (Desktop/Unsupported Browsers)
      console.log('Web Share API unsupported for files, using download fallback.');
      fallbackDownload(blob, safeFilename);
    }
  }, 'image/jpeg', 0.95);
}

/**
 * Fallback to direct client-side file download.
 */
function fallbackDownload(blob, filename) {
  const link = document.createElement('a');
  link.href = URL.createObjectURL(blob);
  link.download = filename;
  document.body.appendChild(link);
  link.click();
  document.body.removeChild(link);
  
  // Alert the user on mobile browsers that download fallback was used
  const isMobile = /iPhone|iPad|iPod|Android/i.test(navigator.userAgent);
  if (isMobile) {
    alert('Sharing directly is unsupported. The image has been downloaded. You can share it from your gallery.');
  }
}
```

    function constructUPIURI(data) {
      const params = new URLSearchParams();
      params.append('pa', data.pa);
      params.append('pn', data.pn);
      if (data.am && parseFloat(data.am) > 0) {
        params.append('am', parseFloat(data.am).toFixed(2));
      }
      params.append('cu', 'INR');
      if (data.tn) {
        params.append('tn', data.tn);
      }
      return `upi://pay?${params.toString()}`;
    }

    async function shareOrDownloadCard(canvas, payeeName) {
      const safeFilename = `${payeeName.replace(/[^a-z0-9]/gi, '_').toLowerCase()}_upi_qr.jpg`;
      canvas.toBlob(async (blob) => {
        if (!blob) return alert('Blob generation failed.');
        const file = new File([blob], safeFilename, { type: 'image/jpeg' });
        const shareData = {
          files: [file],
          title: 'UPI QR Payment',
          text: `Scan to pay to ${payeeName}`
        };

        if (navigator.canShare && navigator.canShare({ files: [file] })) {
          try {
            await navigator.share(shareData);
          } catch (err) {
            if (err.name !== 'AbortError') {
              console.error(err);
              fallbackDownload(blob, safeFilename);
            }
          }
        } else {
          fallbackDownload(blob, safeFilename);
        }
      }, 'image/jpeg', 0.95);
    }

    function fallbackDownload(blob, filename) {
      const link = document.createElement('a');
      link.href = URL.createObjectURL(blob);
      link.download = filename;
      document.body.appendChild(link);
      link.click();
      document.body.removeChild(link);
      if (/iPhone|iPad|iPod|Android/i.test(navigator.userAgent)) {
        alert('Web Share API not supported on this browser. The card image has been downloaded directly.');
      }
    }
